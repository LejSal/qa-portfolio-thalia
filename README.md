# QA Portfolio – Thalia.de

Dieses Repository enthält manuell erstellte QA-Dokumentation, die im Rahmen meiner praktischen Übungen im Bereich Software Testing erstellt wurde.

## Inhalt

### Testfälle
- **Testfall_Beispiel_Thalia**  
  16 manuelle Testfälle für die Webseite **thalia.de**, erstellt zur Überprüfung zentraler Funktionen.
- **Testfall_Login**  
  Manuelle Testfälle für die Login-Funktion, basierend auf definierten Anforderungen.

> Die Testfälle sind im Tabellenformat (Spreadsheet) erstellt, um Übersichtlichkeit und Nachvollziehbarkeit zu gewährleisten.

### Bug Reports
- **Funktionaler Bug Report**  
  Simulation eines Fehlers in der Suchfunktion (z. B. keine Ergebnisse bei gültiger Sucheingabe).
- **UX Bug Report**  
  Analyse der Suchfunktion aus Nutzersicht bei verschiedenen Eingaben  
  (leeres Feld, Zahlen, lange Titel, Sonderzeichen) mit Fokus auf Benutzerfreundlichkeit und Fehlermeldungen.

### SRS
- **SRS_Login**  
  Software Requirements Specification für die Login-Funktion als Grundlage für die Testfallerstellung.

## Methoden & Fähigkeiten
- Manuelles Software Testing
- Black-Box-Testing
- Testfall-Design
- Bug Reporting (funktional & UX)
- Analyse von Benutzeranforderungen
